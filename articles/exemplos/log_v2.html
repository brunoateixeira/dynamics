Log de Auditoria

Documentação AI-friendly e Text-to-SQL para consultas na tabela Log_V2, usada para rastrear ações de usuários que não ficam registradas em outras tabelas.

SQL Server (T-SQL) 1 Exemplo Módulo: Auditoria
Atenção

Avisos de Uso

Log_V2 é uma tabela de uso especial. Recorra a ela apenas quando a informação necessária não estiver disponível em nenhuma outra tabela transacional.
LIMITAÇÕES CRÍTICAS
  • Performance: consultas em Log_V2 são significativamente mais lentas. Sempre filtre por data e filial para reduzir o volume de dados varrido.
  • Ligações por Codigo, não por Ordem: por não haver chaves estrangeiras diretas, ligações com outras tabelas são feitas via .Codigo (texto extraído do campo Detalhe). Isso é menos confiável que ligações por .Ordem.
  • Dados textuais: as informações relevantes estão dentro do campo Detalhe (texto livre). É necessário usar CHARINDEX e SUBSTRING para extrair cada valor.
  • Use como CTE ao combinar com outras queries: se precisar cruzar dados do Log_V2 com outra consulta, isole-o em um CTE para manter performance e legibilidade.

Regra de Negócio

Extração de Dados via CHARINDEX

Como funciona: o campo Detalhe contém texto no formato "... Código 001 (Nome do Produto), Quantidade: 2, R$ Unit.: 15,00 ...". Cada valor é extraído localizando o início do marcador de texto e recortando até o delimitador seguinte.
PADRÃO DE EXTRAÇÃO — CHARINDEX + SUBSTRING
  • Use CHARINDEX('marcador', Detalhe) para localizar a posição do início de cada campo.
  • Use SUBSTRING(Detalhe, início, comprimento) para recortar o valor.
  • Sempre proteja com CASE WHEN CHARINDEX(...) > 0 ... ELSE NULL END para evitar erros em registros que não contêm o campo.
  • Use TRIM() ao redor do resultado para remover espaços residuais.
Marcadores conhecidos no campo Detalhe

Filtros no campo Detalhe: use Detalhe LIKE '%marcador%' para garantir que a linha contém o dado antes de tentar extraí-lo. Isso melhora a performance e evita resultados nulos inesperados.

EXEMPLO 01
Relatório de Produtos Excluídos de Vendas
Lista os itens que foram removidos de movimentações de saída, com funcionário responsável, data, hora, produto, quantidade e valor unitário. Usa Log_V2 como fonte pois Movimento_Prod_Serv não registra quem realizou a exclusão.
auditoria exclusão de produto itens removidos tela de saídas CHARINDEX subquery
AVISOS DESTA CONSULTA
  • A ligação com Prod_Serv é feita via Prod_Serv.Codigo = A.Codigo_Produto (texto), não por Ordem. Isso é uma exceção — não replicar este padrão em outras tabelas.
  • O filtro Tela = 'Tela de Saídas' deve estar no WHERE externo (após a subquery), pois é um campo da subquery, não da tabela original.
  • Se for combinar esta consulta com outras, encapsule-a como CTE para isolar o custo de processamento do Log_V2.

T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

SELECT
    A.Filial,
    A.Data,
    A.Hora,
    A.Funcionario,
    A.Sequencia,
    A.Codigo_Produto,
    Prod_Serv.Nome  AS Nome_Produto,
    A.Quantidade,
    A.Valor_Unitario
FROM (
    SELECT
        Log_V2.Tela,
        Filiais.Codigo                                                    AS Filial,

        -- Código do Produto
        CASE
            WHEN CHARINDEX('Código ', Detalhe) > 0
             AND CHARINDEX('(', Detalhe, CHARINDEX('Código ', Detalhe)) > CHARINDEX('Código ', Detalhe)
            THEN TRIM(
                SUBSTRING(
                    Detalhe,
                    CHARINDEX('Código ', Detalhe) + LEN('Código '),
                    CHARINDEX('(', Detalhe, CHARINDEX('Código ', Detalhe))
                    - (CHARINDEX('Código ', Detalhe) + LEN('Código '))
                )
            )
            ELSE NULL
        END                                                                    AS Codigo_Produto,

        -- Quantidade
        CASE
            WHEN CHARINDEX('Quantidade: ', Detalhe) > 0
             AND CHARINDEX(',', Detalhe, CHARINDEX('Quantidade: ', Detalhe)) > CHARINDEX('Quantidade: ', Detalhe)
            THEN TRIM(
                SUBSTRING(
                    Detalhe,
                    CHARINDEX('Quantidade: ', Detalhe) + 12,
                    CHARINDEX(',', Detalhe, CHARINDEX('Quantidade: ', Detalhe))
                    - (CHARINDEX('Quantidade: ', Detalhe) + 12)
                )
            )
            ELSE NULL
        END                                                                    AS Quantidade,

        -- Valor Unitário
        CASE
            WHEN CHARINDEX('R$ Unit.: ', Detalhe) > 0
             AND CHARINDEX(')', Detalhe, CHARINDEX('R$ Unit.: ', Detalhe)) > CHARINDEX('R$ Unit.: ', Detalhe)
            THEN TRIM(
                SUBSTRING(
                    Detalhe,
                    CHARINDEX('R$ Unit.: ', Detalhe) + 10,
                    CHARINDEX(')', Detalhe, CHARINDEX('R$ Unit.: ', Detalhe))
                    - (CHARINDEX('R$ Unit.: ', Detalhe) + 10)
                )
            )
            ELSE NULL
        END                                                                    AS Valor_Unitario,

        -- Sequência da venda
        CASE
            WHEN CHARINDEX('sequência ', Detalhe) > 0
            THEN TRIM(
                SUBSTRING(
                    Detalhe,
                    CHARINDEX('sequência ', Detalhe) + LEN('sequência '),
                    20
                )
            )
            ELSE NULL
        END                                                                    AS Sequencia,

        CONVERT(VARCHAR(10), Log_V2.Data_Servidor, 103)                    AS Data,
        CONVERT(VARCHAR(10), Log_V2.Data_Servidor, 108)                    AS Hora,
        CONCAT(Funcionarios.Codigo, ' - ', Funcionarios.Apelido)           AS Funcionario

    FROM Log_V2
    INNER JOIN Filiais      ON Filiais.Ordem      = Log_V2.Ordem_Filial
    INNER JOIN Funcionarios ON Funcionarios.Ordem = Log_V2.Ordem_Funcionario
    WHERE
        Detalhe LIKE '%foi marcado como excluído na%'
        AND Detalhe LIKE '%Código %'
        AND Log_V2.Data_Servidor >= @Data_Inicial
        AND Log_V2.Data_Servidor < DATEADD(DAY, 1, @Data_Final)
        AND (Filiais.Ordem = @Filial OR @Filial IS NULL)
) A
INNER JOIN Prod_Serv ON Prod_Serv.Codigo = A.Codigo_Produto  -- ligação por Codigo (exceção)
WHERE
    A.Tela = 'Tela de Saídas'
ORDER BY
    A.Funcionario,
    CONVERT(DATE, A.Data, 103),
    A.Sequencia,
    A.Codigo_Produto
