Filtros Padrão de Produtos

Use sempre este bloco WHERE como base em consultas de produtos. Ele exclui automaticamente itens inativos, bloqueados, de uso interno e registros de sistema (Codigo ≤ '0').
Campos de controle relevantes
CampoTipoRegra
Prod_Serv.InativobitFiltrar = 0 — excluir produtos inativos
Prod_Serv.Bloqueado_MovimentobitFiltrar = 0 — excluir produtos bloqueados para movimentação
Prod_Serv.Bloqueado_VendabitFiltrar = 0 — excluir produtos bloqueados para venda
Prod_Serv.Uso_InternobitFiltrar = 0 — excluir produtos de uso interno
Prod_Serv.CodigovarcharFiltrar > '0' — excluir registros de sistema sem código válido
Prod_Serv.Codigo_OrdenacaovarcharUse no ORDER BY para ordenação correta (não use Codigo diretamente)
T-SQL — WHERE base para produtos
WHERE
    Prod_Serv.Inativo              = 0
    AND Prod_Serv.Bloqueado_Movimento = 0
    AND Prod_Serv.Bloqueado_Venda     = 0
    AND Prod_Serv.Uso_Interno         = 0
    AND Prod_Serv.Codigo              > '0'
ORDER BY
    Prod_Serv.Codigo_Ordenacao
Regra de Negócio

Tabelas de Preço — OUTER APPLY

Não usar JOIN convencional com Prod_Serv_Precos. A tabela Prod_Serv_Precos pode retornar múltiplas linhas por produto. Um INNER JOIN geraria duplicatas. Use sempre OUTER APPLY com filtro de tabela de preço dentro do subselect.
PADRÃO — Um OUTER APPLY por tabela de preço
  • Cada tabela de preço a exibir exige seu próprio bloco OUTER APPLY com alias distinto (Preco_1, Preco_2 etc.).
  • O filtro Tabelas_Preco.Ordem = @Tabela_N deve estar dentro do OUTER APPLY, no INNER JOIN de Tabelas_Preco.
  • Com múltiplos OUTER APPLY, incluir todos os campos no GROUP BY.
  • OUTER APPLY retorna NULL quando o produto não tem preço cadastrado na tabela solicitada — comportamento correto, não é um erro.
T-SQL — Padrão OUTER APPLY para uma tabela de preço
OUTER APPLY (
    SELECT Preco, Nome
    FROM Prod_Serv_Precos
    INNER JOIN Tabelas_Preco
        ON  Tabelas_Preco.Ordem = Prod_Serv_Precos.Ordem_Tabela_Preco
        AND Tabelas_Preco.Ordem = @Tabela_1
    WHERE Prod_Serv_Precos.Ordem_Prod_Serv = Prod_Serv.Ordem
) Preco_1

SELECT
    Prod_Serv.Codigo                                              AS Codigo_Produto,

EXEMPLO 02
Relatório de Preços de Produtos — Uma Tabela
Retorna o preço de cada produto para uma tabela de preço específica. Usa OUTER APPLY para evitar duplicatas.
tabela de preço preços OUTER APPLY Prod_Serv_Precos

Ordem da tabela de preço (Tabelas_Preco.Ordem)

ColunaOrigemDescrição

Nome_ProdutoProd_Serv.NomeNome do produto
Preco_1Preco_1.Preco (OUTER APPLY)Valor do preço na tabela solicitada. NULL se não cadastrado.
Tabela_1Preco_1.Nome (OUTER APPLY)Nome da tabela de preço
T-SQL
DECLARE
    @Tabela_1  INT = 1  -- Ordem da tabela de preço desejada

SELECT
    Prod_Serv.Codigo   AS Codigo_Produto,
    Prod_Serv.Nome     AS Nome_Produto,
    Preco_1.Preco     AS Preco_1,
    Preco_1.Nome      AS Tabela_1
FROM
    Prod_Serv
    OUTER APPLY (
        SELECT Preco, Nome
        FROM Prod_Serv_Precos
        INNER JOIN Tabelas_Preco
            ON  Tabelas_Preco.Ordem = Prod_Serv_Precos.Ordem_Tabela_Preco
            AND Tabelas_Preco.Ordem = @Tabela_1
        WHERE Prod_Serv_Precos.Ordem_Prod_Serv = Prod_Serv.Ordem
    ) Preco_1
WHERE
    Prod_Serv.Inativo              = 0
    AND Prod_Serv.Bloqueado_Movimento = 0
    AND Prod_Serv.Bloqueado_Venda     = 0
    AND Prod_Serv.Uso_Interno         = 0
    AND Prod_Serv.Codigo              > '0'
ORDER BY
    Prod_Serv.Codigo_Ordenacao
EXEMPLO 03
Relatório de Preços de Produtos — Várias Tabelas
Exibe o preço de cada produto em até quatro tabelas de preço simultaneamente, cada uma em sua própria coluna. Requer um OUTER APPLY e alias distinto por tabela, além de GROUP BY com todos os campos.
tabela de preço múltiplas tabelas comparativo de preços OUTER APPLY GROUP BY
REGRA — GROUP BY obrigatório com múltiplos OUTER APPLY
  • Com dois ou mais OUTER APPLY, inclua todos os campos do SELECT no GROUP BY para evitar linhas duplicadas geradas pela combinação dos subselects.
  • Inclua também Prod_Serv.Codigo_Ordenacao no GROUP BY para manter a ordenação correta.
Parâmetros

@Tabela_3INT
Ordem da 3ª tabela de preço
@Tabela_4INT
Ordem da 4ª tabela de preço
Colunas de Saída
ColunaOrigemDescrição
Codigo_ProdutoProd_Serv.CodigoCódigo do produto
Nome_ProdutoProd_Serv.NomeNome do produto
Preco_1 … Preco_4Preco_N.Preco (OUTER APPLY)Valor na tabela N. NULL se não cadastrado.

T-SQL
DECLARE
    @Tabela_1  INT = 1,
    @Tabela_2  INT = 2,
    @Tabela_3  INT = 3,
    @Tabela_4  INT = 4

SELECT
    Prod_Serv.Codigo   AS Codigo_Produto,
    Prod_Serv.Nome     AS Nome_Produto,
    Preco_1.Preco     AS Preco_1,
    Preco_1.Nome      AS Tabela_1,
    Preco_2.Preco     AS Preco_2,
    Preco_2.Nome      AS Tabela_2,
    Preco_3.Preco     AS Preco_3,
    Preco_3.Nome      AS Tabela_3,
    Preco_4.Preco     AS Preco_4,
    Preco_4.Nome      AS Tabela_4
FROM
    Prod_Serv
    OUTER APPLY (
        SELECT Preco, Nome FROM Prod_Serv_Precos
        INNER JOIN Tabelas_Preco ON Tabelas_Preco.Ordem = Prod_Serv_Precos.Ordem_Tabela_Preco AND Tabelas_Preco.Ordem = @Tabela_1
        WHERE Prod_Serv_Precos.Ordem_Prod_Serv = Prod_Serv.Ordem
    ) Preco_1
    OUTER APPLY (
        SELECT Preco, Nome FROM Prod_Serv_Precos
        INNER JOIN Tabelas_Preco ON Tabelas_Preco.Ordem = Prod_Serv_Precos.Ordem_Tabela_Preco AND Tabelas_Preco.Ordem = @Tabela_2
        WHERE Prod_Serv_Precos.Ordem_Prod_Serv = Prod_Serv.Ordem
    ) Preco_2
    OUTER APPLY (
        SELECT Preco, Nome FROM Prod_Serv_Precos
        INNER JOIN Tabelas_Preco ON Tabelas_Preco.Ordem = Prod_Serv_Precos.Ordem_Tabela_Preco AND Tabelas_Preco.Ordem = @Tabela_3
        WHERE Prod_Serv_Precos.Ordem_Prod_Serv = Prod_Serv.Ordem
    ) Preco_3
    OUTER APPLY (
        SELECT Preco, Nome FROM Prod_Serv_Precos
        INNER JOIN Tabelas_Preco ON Tabelas_Preco.Ordem = Prod_Serv_Precos.Ordem_Tabela_Preco AND Tabelas_Preco.Ordem = @Tabela_4
        WHERE Prod_Serv_Precos.Ordem_Prod_Serv = Prod_Serv.Ordem
    ) Preco_4
WHERE
    Prod_Serv.Inativo              = 0
    AND Prod_Serv.Bloqueado_Movimento = 0
    AND Prod_Serv.Bloqueado_Venda     = 0
    AND Prod_Serv.Uso_Interno         = 0
    AND Prod_Serv.Codigo              > '0'
GROUP BY
    Prod_Serv.Codigo_Ordenacao,
    Prod_Serv.Codigo,
    Prod_Serv.Nome,
    Preco_1.Preco, Preco_1.Nome,
    Preco_2.Preco, Preco_2.Nome,
    Preco_3.Preco, Preco_3.Nome,
    Preco_4.Preco, Preco_4.Nome
ORDER BY
    Prod_Serv.Codigo_Ordenacao
