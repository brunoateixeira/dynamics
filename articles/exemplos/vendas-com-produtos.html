Vendas com Produtos

Documentação AI-friendly e Text-to-SQL para consultas que combinam Movimento e Movimento_Prod_Serv, detalhando os produtos vendidos em cada movimentação.

SQL Server (T-SQL) 5 Exemplos Módulo: Vendas / Produtos
Global

Ligações Disponíveis

Use apenas os JOINs necessários para o dado solicitado. As ligações de classificações de produto (Classes, Subclasses etc.) devem ser sempre LEFT JOIN para não excluir produtos sem classificação cadastrada.
TabelaChave de LigaçãoTipo de JOINQuando Usar
Filiais Movimento.Ordem_Filial = Filiais.Ordem INNER JOIN Sempre — para exibir ou filtrar por filial
Movimento_Prod_Serv Movimento.Ordem = Movimento_Prod_Serv.Ordem_Movimento INNER JOIN Sempre — é a tabela de itens da movimentação
Prod_Serv Movimento_Prod_Serv.Ordem_Prod_Serv = Prod_Serv.Ordem INNER JOIN Sempre — para obter código e nome do produto
Funcionarios Movimento_Prod_Serv.Ordem_Vendedor = Funcionarios.Ordem INNER JOIN Quando precisar do vendedor por item. Use Movimento_Prod_Serv.Ordem_Vendedor, não Movimento.Ordem_Vendedor1
Classes Classes.Ordem = Prod_Serv.Ordem_Classe LEFT JOIN Quando precisar filtrar ou exibir a classe do produto
Subclasses Subclasses.Ordem = Prod_Serv.Ordem_Subclasse LEFT JOIN Quando precisar filtrar ou exibir a subclasse do produto
Grupos Grupos.Ordem = Prod_Serv.Ordem_Grupo LEFT JOIN Quando precisar filtrar ou exibir o grupo do produto
Familias Familias.Ordem = Prod_Serv.Ordem_Familia LEFT JOIN Quando precisar filtrar ou exibir a família do produto
Fabricantes Fabricantes.Ordem = Prod_Serv.Ordem_Fabricante LEFT JOIN Quando precisar filtrar ou exibir o fabricante do produto
Regra de Negócio

Campos de Valor em Consultas com Produtos

Atenção: ao listar produtos, os campos de valor devem vir de Movimento_Prod_Serv, não de Movimento. Como a consulta gera múltiplas linhas por movimentação (uma por item), os valores de Movimento seriam duplicados.
CampoTabelaDescrição
Preco_Final_Relatorio Movimento_Prod_Serv Valor líquido do item — campo principal para exibir o valor do produto vendido. Inclui descontos. Use SUM() ao agrupar.
Preco_Total_Sem_Desconto Movimento_Prod_Serv Valor bruto do item, sem descontos aplicados. Use SUM() ao agrupar.
Quantidade Movimento_Prod_Serv Quantidade vendida do item. Use SUM() ao agrupar.
Desconto_Percentual Movimento_Prod_Serv Percentual de desconto já aplicado ao item (ex: 10.00 = 10%). Não requer cálculo adicional. Não usar com SUM() — deve estar no GROUP BY.
Linha_Excluida Movimento_Prod_Serv Indica se o item foi excluído da movimentação. Sempre filtrar Linha_Excluida = 0.
REGRA — Devoluções com valores negativos
  • Ao incluir 'DEV' e 'CVE', aplique o padrão abaixo em todos os campos de valor e quantidade:
SUM(CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF') THEN campo ELSE -campo END)
Regra de Negócio

Classificações de Produto

Classificações são opcionais no cadastro do produto. Use sempre LEFT JOIN para não excluir produtos sem classificação. Nos filtros, sempre inclua o padrão OR @Filtro IS NULL para permitir que o parâmetro seja opcional.
PADRÃO de filtro para classificações
  • Classe: (Prod_Serv.Ordem_Classe = @Classe OR @Classe IS NULL)
  • Subclasse: (Prod_Serv.Ordem_Subclasse = @Subclasse OR @Subclasse IS NULL)
  • Grupo: (Prod_Serv.Ordem_Grupo = @Grupo OR @Grupo IS NULL)
  • Família: (Prod_Serv.Ordem_Familia = @Família OR @Família IS NULL)
  • Fabricante: (Prod_Serv.Ordem_Fabricante = @Fabricante OR @Fabricante IS NULL)
CAMPOS úteis de classificação nas tabelas de lookup
  • Todas as tabelas de classificação possuem .Codigo e .Nome para exibição.
  • A ligação ao produto se dá sempre por Tabela.Ordem = Prod_Serv.Ordem_[Tabela].

EXEMPLO 01
Relatório de Vendas com Produtos
Lista os produtos vendidos no período com quantidade e valor total por produto. Sem abater devoluções.
produtos vendidos quantidade valor por produto ranking de produtos Movimento_Prod_Serv
Observações
NOTA
  • Use SUM(Movimento_Prod_Serv.Preco_Final_Relatorio) — nunca Movimento.Preco_Final_Somado — em consultas que listam produtos.
  • Sempre filtrar Movimento_Prod_Serv.Linha_Excluida = 0 para ignorar itens excluídos da movimentação.
Parâmetros
@Data_Inicial DATE
Data inicial do período (por data de efetivação de estoque)
@Data_Final DATE
Data final do período (por data de efetivação de estoque)
@Filial INT
Ordem da filial
NULL = todas as filiais
Ligações
Movimento→FiliaisON Movimento.Ordem_Filial = Filiais.Ordem
Movimento→Movimento_Prod_ServON Movimento.Ordem = Movimento_Prod_Serv.Ordem_MovimentoItens da venda
Movimento_Prod_Serv→Prod_ServON Movimento_Prod_Serv.Ordem_Prod_Serv = Prod_Serv.OrdemDados do produto
Colunas de Saída
ColunaOrigemDescrição
Codigo_ProdutoProd_Serv.CodigoCódigo do produto no cadastro
Nome_ProdutoProd_Serv.NomeNome do produto
Quantidade_Produto_SomadoSUM(Movimento_Prod_Serv.Quantidade)Quantidade total vendida no período
Valor_Produto_SomadoSUM(Movimento_Prod_Serv.Preco_Final_Relatorio)Valor líquido total vendido no período
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

SELECT
    Prod_Serv.Codigo                                         AS Codigo_Produto,
    Prod_Serv.Nome                                           AS Nome_Produto,
    SUM(Movimento_Prod_Serv.Quantidade)                       AS Quantidade_Produto_Somado,
    SUM(Movimento_Prod_Serv.Preco_Final_Relatorio)            AS Valor_Produto_Somado
FROM
    Movimento
    INNER JOIN Filiais            ON Movimento.Ordem_Filial          = Filiais.Ordem
    INNER JOIN Movimento_Prod_Serv ON Movimento.Ordem               = Movimento_Prod_Serv.Ordem_Movimento
    INNER JOIN Prod_Serv          ON Movimento_Prod_Serv.Ordem_Prod_Serv = Prod_Serv.Ordem
WHERE
    Movimento.Apagado = 0
    AND Movimento.Situacao_Expedicao <> 'A'
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
    AND Movimento.Data_Passou_Efetivacao_Estoque >= @Data_Inicial
    AND Movimento.Data_Passou_Efetivacao_Estoque < DATEADD(DAY, 1, @Data_Final)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF')
    AND (Filiais.Ordem = @Filial OR @Filial IS NULL)
    AND Movimento_Prod_Serv.Linha_Excluida = 0
GROUP BY
    Prod_Serv.Codigo,
    Prod_Serv.Nome
ORDER BY
    Valor_Produto_Somado DESC

Codigo_ProdutoProd_Serv.CodigoCódigo do produto

GROUP BY
    Prod_Serv.Codigo,
    Prod_Serv.Nome
ORDER BY
    Valor_Produto_Somado DESC
EXEMPLO 03
Relatório de Vendas com Produtos e Vendedor
Lista os produtos vendidos agrupados por vendedor. Usa o campo de vendedor do item (Movimento_Prod_Serv.Ordem_Vendedor), que pode diferir do vendedor da movimentação.
vendedor produtos por vendedor comissão Funcionarios Movimento_Prod_Serv
REGRA — Vendedor em consultas com produtos
  • Use Movimento_Prod_Serv.Ordem_Vendedor (não Movimento.Ordem_Vendedor1) para ligar à tabela Funcionarios. O vendedor pode ser diferente por item.
  • Prefira Funcionarios.Apelido ao invés de Funcionarios.Nome pelo tamanho do campo.
Parâmetros
@Data_Inicial DATE
Data inicial do período
@Data_Final DATE
Data final do período
@Filial INT
Ordem da filial
NULL = todas as filiais
@Vendedor INT
Ordem do vendedor (Funcionarios.Ordem)
NULL = todos os vendedores
Ligações
Movimento→FiliaisON Movimento.Ordem_Filial = Filiais.Ordem
Movimento→Movimento_Prod_ServON Movimento.Ordem = Movimento_Prod_Serv.Ordem_Movimento
Movimento_Prod_Serv→Prod_ServON Movimento_Prod_Serv.Ordem_Prod_Serv = Prod_Serv.Ordem
Movimento_Prod_Serv→FuncionariosON Movimento_Prod_Serv.Ordem_Vendedor = Funcionarios.OrdemVendedor do item
Colunas de Saída
ColunaOrigemDescrição
Codigo_FuncionárioFuncionarios.CodigoCódigo do vendedor
Nome_FuncionárioFuncionarios.ApelidoApelido do vendedor
Codigo_ProdutoProd_Serv.CodigoCódigo do produto
Nome_ProdutoProd_Serv.NomeNome do produto

T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL,
    @Vendedor      INT  = NULL  -- NULL = todos os vendedores

SELECT
    Funcionarios.Codigo                                                    AS Codigo_Funcionário,
    Funcionarios.Apelido                                                   AS Nome_Funcionário,
    Prod_Serv.Codigo                                                       AS Codigo_Produto,
    Prod_Serv.Nome                                                         AS Nome_Produto,
    SUM(CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF')
             THEN  Movimento_Prod_Serv.Quantidade
             ELSE -Movimento_Prod_Serv.Quantidade
        END)                                                                   AS Quantidade_Produto_Somado,
    SUM(CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF')
             THEN  Movimento_Prod_Serv.Preco_Final_Relatorio
             ELSE -Movimento_Prod_Serv.Preco_Final_Relatorio
        END)                                                                   AS Valor_Produto_Somado
FROM
    Movimento
    INNER JOIN Filiais            ON Movimento.Ordem_Filial               = Filiais.Ordem
    INNER JOIN Movimento_Prod_Serv ON Movimento.Ordem                    = Movimento_Prod_Serv.Ordem_Movimento
    INNER JOIN Prod_Serv          ON Movimento_Prod_Serv.Ordem_Prod_Serv  = Prod_Serv.Ordem
    INNER JOIN Funcionarios       ON Movimento_Prod_Serv.Ordem_Vendedor   = Funcionarios.Ordem
WHERE
    Movimento.Apagado = 0
    AND Movimento.Situacao_Expedicao <> 'A'
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
    AND Movimento.Data_Passou_Efetivacao_Estoque >= @Data_Inicial
    AND Movimento.Data_Passou_Efetivacao_Estoque < DATEADD(DAY, 1, @Data_Final)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF', 'DEV', 'CVE')
    AND (Filiais.Ordem        = @Filial   OR @Filial   IS NULL)
    AND (Funcionarios.Ordem  = @Vendedor OR @Vendedor IS NULL)
    AND Movimento_Prod_Serv.Linha_Excluida = 0
GROUP BY
    Prod_Serv.Codigo,
    Prod_Serv.Nome,
    Funcionarios.Codigo,
    Funcionarios.Apelido
ORDER BY
    Funcionarios.Codigo,
    Valor_Produto_Somado DESC
EXEMPLO 04
Relatório de Vendas com Classificação de Produtos
Lista os produtos vendidos com dados de classificação (Classe, Subclasse, Grupo, Família, Fabricante). Todos os filtros de classificação são opcionais.
classificação classe subclasse grupo família fabricante LEFT JOIN
REGRA — LEFT JOIN para classificações
  • Use LEFT JOIN em todas as ligações de classificação para incluir produtos sem classificação cadastrada.
  • Todos os parâmetros de classificação aceitam NULL para retornar todos.
Parâmetros
@Data_InicialDATE
Data inicial do período
@Data_FinalDATE
Data final do período
@FilialINT
Ordem da filial
NULL = todas as filiais
@ClasseINT
Ordem da classe do produto
NULL = todas as classes
@SubclasseINT
Ordem da subclasse do produto
NULL = todas as subclasses
@GrupoINT
Ordem do grupo do produto
NULL = todos os grupos
@FamíliaINT
Ordem da família do produto
NULL = todas as famílias
@FabricanteINT
Ordem do fabricante do produto
NULL = todos os fabricantes
Ligações
Movimento→FiliaisON Movimento.Ordem_Filial = Filiais.Ordem
Movimento→Movimento_Prod_ServON Movimento.Ordem = Movimento_Prod_Serv.Ordem_Movimento
Movimento_Prod_Serv→Prod_ServON Movimento_Prod_Serv.Ordem_Prod_Serv = Prod_Serv.Ordem
Prod_Serv→ClassesLEFT JOIN — Classes.Ordem = Prod_Serv.Ordem_Classe
Prod_Serv→SubclassesLEFT JOIN — Subclasses.Ordem = Prod_Serv.Ordem_Subclasse
Prod_Serv→GruposLEFT JOIN — Grupos.Ordem = Prod_Serv.Ordem_Grupo
Prod_Serv→FamiliasLEFT JOIN — Familias.Ordem = Prod_Serv.Ordem_Familia
Prod_Serv→FabricantesLEFT JOIN — Fabricantes.Ordem = Prod_Serv.Ordem_Fabricante
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL,
    @Classe        INT  = NULL,
    @Subclasse     INT  = NULL,
    @Grupo         INT  = NULL,
    @Família       INT  = NULL,
    @Fabricante    INT  = NULL

SELECT
    Prod_Serv.Codigo                                                          AS Codigo_Produto,
    Prod_Serv.Nome                                                            AS Nome_Produto,
    Classes.Codigo                                                            AS Codigo_Classe,
    Classes.Nome                                                              AS Nome_Classe,
    SUM(CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF')
             THEN  Movimento_Prod_Serv.Quantidade
             ELSE -Movimento_Prod_Serv.Quantidade
        END)                                                                     AS Quantidade_Produto_Somado,
    SUM(CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF')
             THEN  Movimento_Prod_Serv.Preco_Final_Relatorio
             ELSE -Movimento_Prod_Serv.Preco_Final_Relatorio
        END)                                                                     AS Valor_Produto_Somado
FROM
    Movimento
    INNER JOIN Filiais            ON Movimento.Ordem_Filial               = Filiais.Ordem
    INNER JOIN Movimento_Prod_Serv ON Movimento.Ordem                    = Movimento_Prod_Serv.Ordem_Movimento
    INNER JOIN Prod_Serv          ON Movimento_Prod_Serv.Ordem_Prod_Serv  = Prod_Serv.Ordem
    LEFT JOIN  Classes            ON Classes.Ordem                     = Prod_Serv.Ordem_Classe
    LEFT JOIN  Subclasses         ON Subclasses.Ordem                  = Prod_Serv.Ordem_Subclasse
    LEFT JOIN  Grupos             ON Grupos.Ordem                      = Prod_Serv.Ordem_Grupo
    LEFT JOIN  Familias           ON Familias.Ordem                    = Prod_Serv.Ordem_Familia
    LEFT JOIN  Fabricantes        ON Fabricantes.Ordem                 = Prod_Serv.Ordem_Fabricante
WHERE
    Movimento.Apagado = 0
    AND Movimento.Situacao_Expedicao <> 'A'
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
    AND Movimento.Data_Passou_Efetivacao_Estoque >= @Data_Inicial
    AND Movimento.Data_Passou_Efetivacao_Estoque < DATEADD(DAY, 1, @Data_Final)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF', 'DEV', 'CVE')
    AND (Filiais.Ordem           = @Filial     OR @Filial     IS NULL)
    AND Movimento_Prod_Serv.Linha_Excluida = 0
    AND (Prod_Serv.Ordem_Classe      = @Classe     OR @Classe     IS NULL)
    AND (Prod_Serv.Ordem_Subclasse   = @Subclasse  OR @Subclasse  IS NULL)
    AND (Prod_Serv.Ordem_Grupo       = @Grupo      OR @Grupo      IS NULL)
    AND (Prod_Serv.Ordem_Familia     = @Família    OR @Família    IS NULL)
    AND (Prod_Serv.Ordem_Fabricante  = @Fabricante OR @Fabricante IS NULL)
GROUP BY
    Prod_Serv.Codigo,
    Prod_Serv.Nome,
    Classes.Codigo,
    Classes.Nome
ORDER BY
    Valor_Produto_Somado DESC
EXEMPLO 05
Relatório de Vendas com Valor Bruto e Valor Líquido por Produto
Exibe o valor bruto (sem desconto), o percentual de desconto aplicado e o valor líquido de cada produto vendido, por número de movimentação.
valor bruto valor líquido desconto percentual de desconto margens Preco_Total_Sem_Desconto
REGRA — Desconto_Percentual não usa SUM()
  • Movimento_Prod_Serv.Desconto_Percentual já contém o percentual aplicado (ex: 10.00 = 10%). Não requer cálculo.
  • Por não ser um valor agregável, deve ser incluído no GROUP BY, não em SUM().
NOTA — Data utilizada neste exemplo
  • Este exemplo filtra por Data_Efetivado_Financeiro (data de efetivação financeira) em vez de Data_Passou_Efetivacao_Estoque. Use conforme o contexto do relatório desejado.
Colunas de Saída
ColunaOrigemDescrição
SequenciaMovimento.SequenciaNúmero da movimentação
Codigo_ProdutoProd_Serv.CodigoCódigo do produto
Nome_ProdutoProd_Serv.NomeNome do produto
Valor_Bruto_ProdutoSUM CASE em Movimento_Prod_Serv.Preco_Total_Sem_DescontoValor sem descontos (negativo em devoluções)
Percentual_Desconto_ProdutoMovimento_Prod_Serv.Desconto_Percentual% de desconto aplicado ao item
Valor_Liquido_ProdutoSUM CASE em Movimento_Prod_Serv.Preco_Final_RelatorioValor com descontos (negativo em devoluções)
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

SELECT
    Movimento.Sequencia,
    Prod_Serv.Codigo                                                          AS Codigo_Produto,
    Prod_Serv.Nome                                                            AS Nome_Produto,
    SUM(CASE WHEN Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF')
             THEN  Movimento_Prod_Serv.Preco_Total_Sem_Desconto
             ELSE -Movimento_Prod_Serv.Preco_Total_Sem_Desconto
        END)                                                                     AS Valor_Bruto_Produto,
    Movimento_Prod_Serv.Desconto_Percentual                                   AS Percentual_Desconto_Produto,
    SUM(CASE WHEN Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF')
             THEN  Movimento_Prod_Serv.Preco_Final_Relatorio
             ELSE -Movimento_Prod_Serv.Preco_Final_Relatorio
        END)                                                                     AS Valor_Liquido_Produto
FROM
    Movimento
    INNER JOIN Movimento_Prod_Serv ON Movimento.Ordem               = Movimento_Prod_Serv.Ordem_Movimento
    INNER JOIN Prod_Serv          ON Movimento_Prod_Serv.Ordem_Prod_Serv = Prod_Serv.Ordem
    INNER JOIN Filiais            ON Movimento.Ordem_Filial           = Filiais.Ordem
WHERE
    Movimento.Apagado = 0
    AND Movimento.Situacao_Expedicao <> 'A'
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
    AND Movimento.Data_Efetivado_Financeiro >= @Data_Inicial
    AND Movimento.Data_Efetivado_Financeiro < DATEADD(DAY, 1, @Data_Final)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF', 'DEV', 'CVE')
    AND Movimento_Prod_Serv.Linha_Excluida = 0
    AND (Filiais.Ordem = @Filial OR @Filial IS NULL)
GROUP BY
    Movimento.Sequencia,
    Prod_Serv.Codigo,
    Prod_Serv.Nome,
    Movimento_Prod_Serv.Desconto_Percentual
ORDER BY
    Movimento.Sequencia
