Ligações Disponíveis

Use apenas os JOINs necessários para o dado solicitado. Incluir JOINs desnecessários pode causar duplicidade de registros.

Filtro Padrão de Recebimentos Baixados

Use este bloco WHERE como base em todas as consultas de recebimentos. Ele garante que apenas contas a receber válidas, quitadas e sem duplicidade sejam retornadas.
REGRA — Parâmetro de filial opcional
  • Quando @Filial é NULL → retorna todas as filiais.
  • Quando @Filial é um inteiro → filtra pela filial específica.
  • Use a variante IS NULL para consistência no padrão desta documentação.
T-SQL — WHERE base para recebimentos 
-- ─── PARÂMETROS ───────────────────────────────────────────────────
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL  -- NULL = todas as filiais

-- ─── WHERE BASE ────────────────────────────────────────────────────
WHERE
    -- 1. Recebimentos no período pela data de quitação
    Financeiro_Contas.Data_Quitacao >= @Data_Inicial
    AND Financeiro_Contas.Data_Quitacao < DATEADD(D, 1, @Data_Final)

    -- 2. Somente contas a receber
    AND Financeiro_Contas.Pagar_Receber = 'R'

    -- 3. Tipos de conta válidos para recebimento
    AND Financeiro_Contas.Tipo_Conta IN ('A','B','C','D','O','N','H','R')

    -- 4. Situações de baixa válidas
    AND Financeiro_Contas.Situacao IN ('F','Q','I','X')

    -- 5. Excluir trocas de produto (não são recebimento financeiro)
    AND Financeiro_Contas.Tipo_Recebido_Pago <> 'T'

    -- 6. Filtro de filial (NULL = todas)
    AND (Financeiro_Contas.Ordem_Filial = @Filial OR @Filial IS NULL)

    -- 7. Regra anti-duplicidade para Tipo_Conta = 'A' (ver seção abaixo)
    AND (
        Financeiro_Contas.Tipo_Conta NOT IN ('A')
        OR (
            Financeiro_Contas.Tipo_Conta IN ('A')
            AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
            AND Financeiro_Contas.Ordem_Pai IS NOT NULL
            AND EXISTS (
                SELECT 1 FROM Financeiro_Contas FC
                WHERE FC.Ordem_Renegociado_Agrupado = Financeiro_Contas.Ordem_Pai
                AND FC.Tipo_Conta IN ('B','C','D','O','N','H','R')
            )
        )
        OR (
            Financeiro_Contas.Tipo_Conta IN ('A')
            AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
            AND Financeiro_Contas.Tela_Origem = 15 -- PDV
        )
    )
Regra de Negócio

Contas Agrupadas (Tipo_Conta = 'A')

Crítico para evitar duplicidade: Contas do tipo 'A' são registros agrupadores. Se incluídas sem filtro, o valor aparecerá duplicado junto com os registros filhos.
CONTEXTO
  • Quando um cliente paga múltiplas parcelas juntas, o sistema gera um registro pai do tipo 'A' e mantém os registros filhos com os tipos originais.
  • Incluir ambos somaria o valor duas vezes.
QUANDO incluir Tipo_Conta = 'A'
  • Cenário 1 — Renegociação via crédito: O registro 'A' foi recebido via Crédito (Tipo_Recebido_Pago = 'C'), possui um Ordem_Pai válido, e existe ao menos um filho renegociado do tipo boleto/cartão/cheque etc.
  • Cenário 2 — PDV: O registro 'A' foi gerado diretamente pelo PDV (Tela_Origem = 15) e recebido via Crédito.
QUANDO excluir Tipo_Conta = 'A'
  • Em todos os demais casos — o filtro padrão já trata isso com a cláusula NOT IN ('A') como caminho padrão.

EXEMPLO 01
Relatório de Recebíveis Baixados
Base para qualquer relatório de recebimentos. Exibe valores e datas das contas quitadas no período, filtradas pela data de quitação.
recebimentos baixas quitações fluxo de caixa relatório financeiro período
Parâmetros
@Data_Inicial DATE
Data inicial do período (por data de quitação)
@Data_Final DATE
Data final do período (por data de quitação)
@Filial INT
Ordem da filial
NULL = todas as filiais
Ligações
Financeiro_Contas → Filiais ON Financeiro_Contas.Ordem_Filial = Filiais.Ordem Exibir nome da filial
Colunas de Saída
ColunaOrigemDescrição
Data_EmissaoFinanceiro_Contas.Data_EmissaoData de emissão da conta
Data_VencimentoFinanceiro_Contas.Data_VencimentoData de vencimento original
Data_QuitacaoFinanceiro_Contas.Data_QuitacaoData efetiva do recebimento
Valor_BrutoFinanceiro_Contas.Valor_BaseValor original sem acréscimos ou descontos
Valor_LiquidoValor_Base + Juros + Multa − DescontoValor efetivo a receber, com todos os ajustes
Valor_QuitadoFinanceiro_Contas.Valor_QuitadoValor que foi efetivamente recebido na baixa
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

SELECT
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Emissao,   103) AS Data_Emissao,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Vencimento, 103) AS Data_Vencimento,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Quitacao,   103) AS Data_Quitacao,
    Financeiro_Contas.Valor_Base                                       AS Valor_Bruto,
    Financeiro_Contas.Valor_Base
        + Financeiro_Contas.Valor_Juros
        + Financeiro_Contas.Valor_Multa
        - Financeiro_Contas.Valor_Desconto                               AS Valor_Liquido,
    Financeiro_Contas.Valor_Quitado                                    AS Valor_Quitado
FROM
    Financeiro_Contas
    INNER JOIN Filiais ON Financeiro_Contas.Ordem_Filial = Filiais.Ordem
WHERE
    Financeiro_Contas.Data_Quitacao >= @Data_Inicial
    AND Financeiro_Contas.Data_Quitacao < DATEADD(D, 1, @Data_Final)
    AND Financeiro_Contas.Pagar_Receber = 'R'
    AND Financeiro_Contas.Tipo_Conta IN ('A','B','C','D','O','N','H','R')
    AND Financeiro_Contas.Situacao IN ('F','Q','I','X')
    AND Financeiro_Contas.Tipo_Recebido_Pago <> 'T'
    AND (Financeiro_Contas.Ordem_Filial = @Filial OR @Filial IS NULL)
    AND (
        Financeiro_Contas.Tipo_Conta NOT IN ('A')
        OR (Financeiro_Contas.Tipo_Conta = 'A' AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
             AND Financeiro_Contas.Ordem_Pai IS NOT NULL
             AND EXISTS (SELECT 1 FROM Financeiro_Contas FC
                         WHERE FC.Ordem_Renegociado_Agrupado = Financeiro_Contas.Ordem_Pai
                         AND FC.Tipo_Conta IN ('B','C','D','O','N','H','R')))
        OR (Financeiro_Contas.Tipo_Conta = 'A' AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
             AND Financeiro_Contas.Tela_Origem = 15)
    )
ORDER BY
    CONVERT(DATE, Financeiro_Contas.Data_Quitacao),
    CONVERT(DATE, Financeiro_Contas.Data_Emissao)

    AND (Financeiro_Contas.Ordem_Filial = @Filial OR @Filial IS NULL)
    AND (
        Financeiro_Contas.Tipo_Conta NOT IN ('A')
        OR (Financeiro_Contas.Tipo_Conta = 'A' AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
             AND Financeiro_Contas.Ordem_Pai IS NOT NULL

DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL,
    @Cliente       INT  = NULL  -- NULL = todos os clientes

SELECT
    Cli_For.Codigo                                                       AS Codigo_Cliente,
    Cli_For.Nome                                                         AS Nome_Cliente,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Emissao,   103)          AS Data_Emissao,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Vencimento, 103)          AS Data_Vencimento,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Quitacao,   103)          AS Data_Quitacao,
    Financeiro_Contas.Valor_Base                                          AS Valor_Bruto,
    Financeiro_Contas.Valor_Base
        + Financeiro_Contas.Valor_Juros
        + Financeiro_Contas.Valor_Multa
        - Financeiro_Contas.Valor_Desconto                                  AS Valor_Liquido,
    Financeiro_Contas.Valor_Quitado                                       AS Valor_Quitado
FROM
    Financeiro_Contas
    INNER JOIN Filiais  ON Financeiro_Contas.Ordem_Filial  = Filiais.Ordem
    INNER JOIN Cli_For  ON Cli_For.Ordem              = Financeiro_Contas.Ordem_Cli_For
WHERE
    Financeiro_Contas.Data_Quitacao >= @Data_Inicial
    AND Financeiro_Contas.Data_Quitacao < DATEADD(D, 1, @Data_Final)
    AND Financeiro_Contas.Pagar_Receber = 'R'
    AND Financeiro_Contas.Tipo_Conta IN ('A','B','C','D','O','N','H','R')
    AND Financeiro_Contas.Situacao IN ('F','Q','I','X')
    AND Financeiro_Contas.Tipo_Recebido_Pago <> 'T'
    AND (Financeiro_Contas.Ordem_Filial  = @Filial   OR @Filial   IS NULL)
    AND (Financeiro_Contas.Ordem_Cli_For = @Cliente  OR @Cliente  IS NULL)
    AND (
        Financeiro_Contas.Tipo_Conta NOT IN ('A')
        OR (Financeiro_Contas.Tipo_Conta = 'A' AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
             AND Financeiro_Contas.Ordem_Pai IS NOT NULL
             AND EXISTS (SELECT 1 FROM Financeiro_Contas FC
                         WHERE FC.Ordem_Renegociado_Agrupado = Financeiro_Contas.Ordem_Pai
                         AND FC.Tipo_Conta IN ('B','C','D','O','N','H','R')))
        OR (Financeiro_Contas.Tipo_Conta = 'A' AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
             AND Financeiro_Contas.Tela_Origem = 15)
    )
ORDER BY
    Cli_For.Codigo,
    CONVERT(DATE, Financeiro_Contas.Data_Quitacao)
EXEMPLO 05
Recebíveis com Vendedor e Comissão
Exibe os recebimentos com dados do vendedor responsável, incluindo o percentual de comissão cadastrado e o valor calculado da comissão.
vendedor comissão funcionários comissão vendedor recebimentos por vendedor Funcionarios
REGRA — Cálculo de Comissão
  • O campo Funcionarios.Comissao_Venda armazena o percentual (ex: 5.00 = 5%).
  • Fórmula: Valor_Quitado × (Comissao_Venda / 100.0)
  • Use NULLIF(..., 0) para evitar divisão por zero e ISNULL(..., 0) para tratar nulos.
Parâmetros
@Data_Inicial DATE
Data inicial do período
@Data_Final DATE
Data final do período
@Filial INT
Ordem da filial
NULL = todas as filiais
@Vendedor INT
Ordem do vendedor (Funcionarios.Ordem)
NULL = todos os vendedores
Ligações
Financeiro_Contas → Filiais ON Financeiro_Contas.Ordem_Filial = Filiais.Ordem
Financeiro_Contas → Funcionarios ON Funcionarios.Ordem = Financeiro_Contas.Ordem_Vendedor1 Vendedor + comissão
Colunas de Saída
ColunaOrigemDescrição
Codigo_VendedorFuncionarios.CodigoCódigo do vendedor no cadastro
Nome_VendedorFuncionarios.ApelidoNome/apelido do vendedor
Data_EmissaoFinanceiro_Contas.Data_EmissaoData de emissão
Data_VencimentoFinanceiro_Contas.Data_VencimentoData de vencimento
Data_QuitacaoFinanceiro_Contas.Data_QuitacaoData de quitação
Valor_BrutoFinanceiro_Contas.Valor_BaseValor original sem ajustes
Valor_LiquidoValor_Base + Juros + Multa − DescontoValor com todos os ajustes
Valor_QuitadoFinanceiro_Contas.Valor_QuitadoValor efetivamente recebido
Percentual_ComissaoFuncionarios.Comissao_Venda% de comissão do vendedor (ex: 5.00 = 5%)

T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL,
    @Vendedor      INT  = NULL  -- NULL = todos os vendedores

SELECT
    Funcionarios.Codigo                                                  AS Codigo_Vendedor,
    Funcionarios.Apelido                                                 AS Nome_Vendedor,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Emissao,   103)          AS Data_Emissao,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Vencimento, 103)          AS Data_Vencimento,
    CONVERT(VARCHAR(10), Financeiro_Contas.Data_Quitacao,   103)          AS Data_Quitacao,
    Financeiro_Contas.Valor_Base                                          AS Valor_Bruto,
    Financeiro_Contas.Valor_Base
        + Financeiro_Contas.Valor_Juros
        + Financeiro_Contas.Valor_Multa
        - Financeiro_Contas.Valor_Desconto                                  AS Valor_Liquido,
    Financeiro_Contas.Valor_Quitado                                       AS Valor_Quitado,
    Funcionarios.Comissao_Venda                                          AS Percentual_Comissao,
    ISNULL(
        Financeiro_Contas.Valor_Quitado * (NULLIF(Funcionarios.Comissao_Venda, 0) / 100.0),
    0)                                                                      AS Valor_Comissao
FROM
    Financeiro_Contas
    INNER JOIN Filiais      ON Financeiro_Contas.Ordem_Filial    = Filiais.Ordem
    INNER JOIN Funcionarios ON Funcionarios.Ordem             = Financeiro_Contas.Ordem_Vendedor1
WHERE
    Financeiro_Contas.Data_Quitacao >= @Data_Inicial
    AND Financeiro_Contas.Data_Quitacao < DATEADD(D, 1, @Data_Final)
    AND Financeiro_Contas.Pagar_Receber = 'R'
    AND Financeiro_Contas.Tipo_Conta IN ('A','B','C','D','O','N','H','R')
    AND Financeiro_Contas.Situacao IN ('F','Q','I','X')
    AND Financeiro_Contas.Tipo_Recebido_Pago <> 'T'
    AND (Financeiro_Contas.Ordem_Filial  = @Filial   OR @Filial   IS NULL)
    AND (Funcionarios.Ordem            = @Vendedor OR @Vendedor IS NULL)
    AND (
        Financeiro_Contas.Tipo_Conta NOT IN ('A')
        OR (Financeiro_Contas.Tipo_Conta = 'A' AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
             AND Financeiro_Contas.Ordem_Pai IS NOT NULL
             AND EXISTS (SELECT 1 FROM Financeiro_Contas FC
                         WHERE FC.Ordem_Renegociado_Agrupado = Financeiro_Contas.Ordem_Pai
                         AND FC.Tipo_Conta IN ('B','C','D','O','N','H','R')))
        OR (Financeiro_Contas.Tipo_Conta = 'A' AND Financeiro_Contas.Tipo_Recebido_Pago = 'C'
             AND Financeiro_Contas.Tela_Origem = 15)
