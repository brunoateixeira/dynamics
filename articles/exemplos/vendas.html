ENUMs e Valores das Colunas

Para uso em Text-to-SQL: sempre referencie estes mapeamentos ao gerar filtros ou CASE WHEN envolvendo as colunas abaixo. Nunca assuma valores não listados aqui.
Movimento.Tipo_Operacao
Atenção: use sempre IN ('VND','VPC','VEF') para filtrar apenas vendas. Inclua 'DEV' e 'CVE' quando for necessário abater devoluções.
'VND'Venda Normal
'VPC'Venda de Produtos Consignados
'VEF'Venda de Entrega Futura
'DEV'⚠ Devolução (valor negativo)
'CVE'⚠ Cancelamento de Venda de VEF (valor negativo)
Movimento.Situacao_Expedicao
Sempre excluir Situacao_Expedicao = 'A' (Aguardando) das consultas de vendas efetivadas. Incluir apenas quando solicitado.
'A'⛔ Aguardando / Em Aberto (excluir)
'C'Expedição Concluída
Movimento.Apagado
0Ativo — incluir na consulta
1⛔ Apagado — sempre excluir

Ligações Disponíveis

Use apenas os JOINs necessários para o dado solicitado. Incluir JOINs desnecessários pode causar duplicidade de registros.
Tabela Chave de Ligação Quando Usar Colunas Úteis
Filiais Movimento.Ordem_Filial = Filiais.Ordem Quando precisar exibir ou filtrar por filial Filiais.Nome, Filiais.Codigo, Filiais.Razao_Social, Filiais.Endereco, Filiais.Bairro, Filiais.Fone, Filiais.Cidade, Filiais.UF, Filiais.CEP, Filiais.CNPJ, Filiais.Inscricao_Estadual, Filiais.CRT

Filtro Padrão de Vendas

Use este bloco WHERE como base em todas as consultas de vendas. Ele garante que apenas movimentações efetivadas, válidas e sem duplicidade sejam retornadas.
REGRA — Parâmetro de filial opcional
  • Quando @Filial é NULL → retorna todas as filiais.
  • Quando @Filial é um inteiro → filtra pela filial específica.
REGRA — Tipo_Operacao para vendas sem devolução
  • Use IN ('VND','VPC','VEF') para relatórios de vendas brutas.
  • Adicione 'DEV', 'CVE' quando o relatório deve abater devoluções. Nesses casos, aplique CASE WHEN ... THEN valor ELSE -valor END para que devoluções entrem como valores negativos.
T-SQL — WHERE base para vendas (sem devolução) 
-- ─── PARÂMETROS ───────────────────────────────────────────────────
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL  -- NULL = todas as filiais

-- ─── WHERE BASE ────────────────────────────────────────────────────
WHERE
    -- 1. Excluir movimentos apagados
    Movimento.Apagado = 0

    -- 2. Excluir movimentos em expedição pendente
    AND Movimento.Situacao_Expedicao <> 'A'

    -- 3. Excluir movimentos que foram desefetivados do estoque
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL

    -- 4. Filtrar pelo período (data de efetivação de estoque/financeiro)
    AND Movimento.Data_Passou_Efetivacao_Estoque >= @Data_Inicial
    AND Movimento.Data_Passou_Efetivacao_Estoque < DATEADD(DAY, 1, @Data_Final)

    -- 5. Apenas vendas (sem devoluções)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF')

    -- 6. Filtro de filial (NULL = todas)
    AND (Filiais.Ordem = @Filial OR @Filial IS NULL)
Regra de Negócio

Campos de Data

Prioridade de campos de data: use sempre Data_Passou_Efetivacao_Estoque como referência temporal em relatórios de vendas. Ela considera a data de efetivação de estoque e, quando não houver, a de efetivação financeira.
QUANDO usar cada campo de data
  • Relatórios de Vendas: use Data_Passou_Efetivacao_Estoque — campo prioritário para filtrar e exibir datas de venda.
  • Relatórios de Financeiro/Recebimento: use Data_Efetivado_Financeiro.
  • Relatórios de Orçamentos / Movimentos não efetivados: use Data (data da última gravação).
FORMATAÇÃO — Converter datas para DD/MM/AAAA
  • Sempre use CONVERT(VARCHAR(10), Campo_de_Data, 103) para exibir datas no formato brasileiro.
Regra de Negócio

Campos de Valor

Regra de Três Versões: alguns campos de valor existem em três variantes (Total Geral, só Produtos, só Serviços). Sempre priorize o campo que soma Produtos e Serviços.
Campo PrioritárioDescriçãoNão Usar Quando
Preco_Final_Somado Valor total da movimentação — líquido, com impostos e descontos. Use para somar o total de vendas em relatórios sem detalhe de produtos. —
Preco_Total_Sem_Desconto_Somado Valor bruto — total de produtos e serviços sem desconto. —
Desconto_Valor_Somado Desconto total da movimentação (produtos + serviços). Evitar Desconto_Total_Prod ou Desconto_Total_Serv isoladamente.
Frete_Valor_Somado Frete incluso no total da movimentação. Não confundir com Frete_Valor_Separado_Somado (pago à parte).
Qtde_Total_Geral Quantidade total de itens (produtos + serviços). Evitar Qtde_Total_Prod ou Qtde_Total_Serv isoladamente.
Devoluções: ao incluir 'DEV' e 'CVE', aplique CASE WHEN Tipo_Operacao IN ('VND','VPC','VEF') THEN valor ELSE -valor END nos campos de valor para que devoluções entrem como negativos no total.

EXEMPLO 01
Relatório de Vendas
Base para qualquer relatório de vendas. Exibe totais e datas das movimentações efetivadas no período, sem abater devoluções.
vendas total de vendas efetivadas fluxo de caixa relatório financeiro período
Observações
NOTA
  • Sempre utilize Movimento.Preco_Final_Somado para exibir o total da venda em relatórios que não exibem produtos individualmente.
Parâmetros
@Data_Inicial DATE
Data inicial do período (por data de efetivação de estoque)
@Data_Final DATE
Data final do período (por data de efetivação de estoque)
@Filial INT
Ordem da filial
NULL = todas as filiais
Ligações
Movimento → Filiais ON Movimento.Ordem_Filial = Filiais.Ordem Exibir/filtrar filial
Colunas de Saída
ColunaOrigemDescrição
Data_CriaçãoMovimento.DataData da última gravação do movimento
Data_EfetivaçãoMovimento.Data_Passou_Efetivacao_EstoqueData efetiva da venda (campo prioritário)
SequenciaMovimento.SequenciaNúmero identificador da movimentação
Qtde_ProdutosMovimento.Qtde_Total_ProdQuantidade total de itens da movimentação
TotalMovimento.Preco_Final_SomadoValor líquido total da movimentação
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

SELECT
    CONVERT(VARCHAR(10), Movimento.Data,                             103) AS Data_Criação,
    CONVERT(VARCHAR(10), Movimento.Data_Passou_Efetivacao_Estoque,   103) AS Data_Efetivação,
    Movimento.Sequencia,
    Movimento.Qtde_Total_Prod        AS Qtde_Produtos,
    Movimento.Preco_Final_Somado     AS Total
FROM
    Movimento
    INNER JOIN Filiais ON Movimento.Ordem_Filial = Filiais.Ordem
WHERE
    Movimento.Apagado = 0
    AND Movimento.Situacao_Expedicao <> 'A'
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
    AND Movimento.Data_Passou_Efetivacao_Estoque >= @Data_Inicial
    AND Movimento.Data_Passou_Efetivacao_Estoque < DATEADD(DAY, 1, @Data_Final)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF')
    AND (Filiais.Ordem = @Filial OR @Filial IS NULL)
EXEMPLO 02
Relatório de Vendas com Devolução
Retorna as movimentações de venda abatendo o valor de devoluções. Movimentações do tipo DEV e CVE aparecem com valores negativos.
vendas devolução cancelamento venda líquida desconto total frete
REGRA — Valores negativos para devoluções
  • As operações 'DEV' e 'CVE' são incluídas no Tipo_Operacao IN (...).
  • Use CASE WHEN Tipo_Operacao IN ('VND','VPC','VEF') THEN valor ELSE -valor END para todos os campos de valor.
Colunas de Saída
ColunaOrigemDescrição
Data_EfetivaçãoMovimento.Data_Passou_Efetivacao_EstoqueData efetiva da venda ou devolução
SequenciaMovimento.SequenciaNúmero da movimentação
Desconto_TotalCASE em Movimento.Desconto_Total_GeralDesconto total (negativo em devoluções)
Frete_TotalCASE em Movimento.Frete_Valor_SomadoValor de frete (negativo em devoluções)
TotalCASE em Movimento.Preco_Final_SomadoValor total (negativo em devoluções)
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

SELECT
    CONVERT(VARCHAR(10), Movimento.Data_Passou_Efetivacao_Estoque, 103) AS Data_Efetivação,
    Movimento.Sequencia,
    CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF')
         THEN  Movimento.Desconto_Total_Geral
         ELSE -Movimento.Desconto_Total_Geral
    END AS Desconto_Total,
    CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF')
         THEN  Movimento.Frete_Valor_Somado
         ELSE -Movimento.Frete_Valor_Somado
    END AS Frete_Total,
    CASE WHEN Movimento.Tipo_Operacao IN ('VND','VPC','VEF')
         THEN  Movimento.Preco_Final_Somado
         ELSE -Movimento.Preco_Final_Somado
    END AS Total
FROM
    Movimento
    INNER JOIN Filiais ON Movimento.Ordem_Filial = Filiais.Ordem
WHERE
    Movimento.Apagado = 0
    AND Movimento.Situacao_Expedicao <> 'A'
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
    AND Movimento.Data_Passou_Efetivacao_Estoque >= @Data_Inicial
    AND Movimento.Data_Passou_Efetivacao_Estoque < DATEADD(DAY, 1, @Data_Final)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF', 'DEV', 'CVE')
    AND (Filiais.Ordem = @Filial OR @Filial IS NULL)

DECLARE

EXEMPLO 05
Relatório de Vendas por Tipo de Recebimento
Exibe como cada venda foi recebida e o valor por tipo de recebimento. Cada forma de recebimento é tratada como uma linha distinta via CROSS APPLY.
forma de pagamento pix dinheiro cartão vale CROSS APPLY Financeiro_Formas_Recebimento
TÉCNICA — CROSS APPLY para transformar colunas em linhas
  • A tabela Financeiro_Formas_Recebimento_Pagamento_V2 contém uma única linha por venda, com os valores de recebimento em colunas distintas.
  • Use CROSS APPLY (VALUES ...) para transformar essas colunas em linhas.
  • Sempre incluir AND R.Valor IS NOT NULL AND R.Valor <> 0 para exibir apenas os recebimentos utilizados.
Formas de Recebimento Disponíveis
Dinheiro_ValorDinheiro
Conta_Bancaria_ValorPix / Depósito
Cartao_Credito_ValorCartão Crédito
Cartao_Debito_ValorCartão Débito
Cartao_Convenio_ValorCartão Convênio
Cheque_Pre_ValorCheque Pré
Cheque_ValorCheque
Parcelado_ValorParcelado
Credito_ValorCrédito
Vale_ValorVale
Conta_Caderno_ValorConta Caderno
Valor_TrocoTroco
Ligações
Movimento→Cli_For ON Cli_For.Ordem = Movimento.Ordem_Cli_For
Movimento→Funcionarios ON Funcionarios.Ordem = Movimento.Ordem_Vendedor1
Movimento→Filiais ON Filiais.Ordem = Movimento.Ordem_Filial
Movimento→Financeiro_Formas_Recebimento_Pagamento_V2 ON ...Ordem_Movimento = Movimento.Ordem Valores por forma
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

SELECT
    Cli_For.Codigo          AS Codigo_Cliente,
    Cli_For.Nome            AS Nome_Cliente,
    Funcionarios.Apelido    AS Nome_Vendedor,
    Filiais.Codigo          AS Codigo_Filial,
    Movimento.Sequencia     AS Sequência_de_Venda,
    R.Tipo_Recebimento,
    R.Valor
FROM Movimento
INNER JOIN Cli_For       ON Cli_For.Ordem       = Movimento.Ordem_Cli_For
INNER JOIN Funcionarios  ON Funcionarios.Ordem  = Movimento.Ordem_Vendedor1
INNER JOIN Filiais       ON Filiais.Ordem       = Movimento.Ordem_Filial
INNER JOIN Financeiro_Formas_Recebimento_Pagamento_V2
    ON Financeiro_Formas_Recebimento_Pagamento_V2.Ordem_Movimento = Movimento.Ordem
CROSS APPLY (VALUES
    ('Dinheiro',          Financeiro_Formas_Recebimento_Pagamento_V2.Dinheiro_Valor),
    ('Parcelado',         Financeiro_Formas_Recebimento_Pagamento_V2.Parcelado_Valor),
    ('Cartão Crédito',    Financeiro_Formas_Recebimento_Pagamento_V2.Cartao_Credito_Valor),
    ('Cartão Débito',     Financeiro_Formas_Recebimento_Pagamento_V2.Cartao_Debito_Valor),
    ('Cartão Convênio',   Financeiro_Formas_Recebimento_Pagamento_V2.Cartao_Convenio_Valor),
    ('Cheque Pré',        Financeiro_Formas_Recebimento_Pagamento_V2.Cheque_Pre_Valor),
    ('Cheque',            Financeiro_Formas_Recebimento_Pagamento_V2.Cheque_Valor),
    ('Pix',               Financeiro_Formas_Recebimento_Pagamento_V2.Conta_Bancaria_Valor),
    ('Conta Caderno',     Financeiro_Formas_Recebimento_Pagamento_V2.Conta_Caderno_Valor),
    ('Crédito',           Financeiro_Formas_Recebimento_Pagamento_V2.Credito_Valor),
    ('Não Identificado',  Financeiro_Formas_Recebimento_Pagamento_V2.Nao_Identificado_Valor),
    ('Produtos Trocados', Financeiro_Formas_Recebimento_Pagamento_V2.Produtos_Trocados_Valor),
    ('Vale',              Financeiro_Formas_Recebimento_Pagamento_V2.Vale_Valor),
    ('Troco',             Financeiro_Formas_Recebimento_Pagamento_V2.Valor_Troco)
) AS R(Tipo_Recebimento, Valor)
WHERE
    Movimento.Apagado = 0
    AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
    AND Movimento.Data_Efetivado_Financeiro >= @Data_Inicial
    AND Movimento.Data_Efetivado_Financeiro < DATEADD(D, 1, @Data_Final)
    AND (Movimento.Ordem_Filial = @Filial OR @Filial IS NULL)
    AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF', 'DEV', 'CVE')
    -- Exibir apenas formas de recebimento utilizadas
    AND R.Valor IS NOT NULL
    AND R.Valor <> 0
GROUP BY
    Cli_For.Codigo, Cli_For.Nome,
    Funcionarios.Apelido,
    Filiais.Codigo,
    Movimento.Sequencia,
    R.Tipo_Recebimento, R.Valor
ORDER BY
    Movimento.Sequencia, Filiais.Codigo, Cli_For.Codigo
EXEMPLO 06
Relatório de Vendas por Tipo de Recebimento — Detalhado por Parcela
Semelhante ao Exemplo 05, porém detalha os parcelamentos (boleto, cartão de crédito etc.) com o tipo e a quantidade de parcelas. Utiliza CTE + CROSS APPLY para combinar as duas estruturas de tabela.
parcelas parcelamento boleto cartão crédito CTE CROSS APPLY Financeiro_Formas_Recebimento_Contas
TÉCNICA — CTE + CROSS APPLY para detalhar parcelas
  • Financeiro_Formas_Recebimento_Pagamento_V2: uma linha por venda, valores em colunas — usada para recebimentos não-parcelados.
  • Financeiro_Formas_Recebimento_Pagamento_Contas_V2: uma linha por parcela — usada para detalhar boleto, cartão de crédito, carnê, cheque e carteira.
  • O CTE VENDAS une a movimentação à primeira tabela; o CTE CONTAS extrai as parcelas da segunda; o SELECT final combina ambos via LEFT JOIN + CROSS APPLY.
Tipos de forma de recebimento parcelável (Financeiro_Formas_Recebimento_Pagamento_Contas_V2.Tipo_Forma_Recebimento_Pagamento)
  • 'B' → Boleto
  • 'C' → Cartão de Crédito
  • 'D' → Cartão de Débito
  • 'H' → Cheque
  • 'N' → Carnê
  • 'R' → Carteira
Ligações
Movimento→Cli_For ON Cli_For.Ordem = Movimento.Ordem_Cli_For
Movimento→Funcionarios ON Funcionarios.Ordem = Movimento.Ordem_Vendedor1
Movimento→Filiais ON Filiais.Ordem = Movimento.Ordem_Filial
Movimento→Financeiro_Formas_Recebimento_Pagamento_V2 ON ...Ordem_Movimento = Movimento.Ordem
Financeiro_Formas_Recebimento_Pagamento_Contas_V2→VENDAS (CTE) ON ...Ordem_Formas_Recebimento_Pagamento = VENDAS.Ordem Parcelas
T-SQL
DECLARE
    @Data_Inicial  DATE = '2026-01-01',
    @Data_Final    DATE = '2026-03-31',
    @Filial        INT  = NULL

;WITH VENDAS AS (
    SELECT
        Cli_For.Codigo          AS Codigo_Cliente,
        Cli_For.Nome            AS Nome_Cliente,
        Funcionarios.Apelido    AS Nome_Vendedor,
        Filiais.Codigo          AS Codigo_Filial,
        Movimento.Sequencia     AS Sequência_de_Venda,
        Financeiro_Formas_Recebimento_Pagamento_V2.Ordem
            AS Ordem_Financeiro_Formas_Recebimento_Pagamento_V2,
        Financeiro_Formas_Recebimento_Pagamento_V2.*
    FROM Movimento
    INNER JOIN Cli_For      ON Cli_For.Ordem      = Movimento.Ordem_Cli_For
    INNER JOIN Funcionarios ON Funcionarios.Ordem = Movimento.Ordem_Vendedor1
    INNER JOIN Filiais      ON Filiais.Ordem      = Movimento.Ordem_Filial
    INNER JOIN Financeiro_Formas_Recebimento_Pagamento_V2
        ON Financeiro_Formas_Recebimento_Pagamento_V2.Ordem_Movimento = Movimento.Ordem
    WHERE
        Movimento.Apagado = 0
        AND Movimento.Data_Passou_Desefetivacao_Estoque IS NULL
        AND Movimento.Data_Efetivado_Financeiro >= @Data_Inicial
        AND Movimento.Data_Efetivado_Financeiro < DATEADD(D, 1, @Data_Final)
        AND (Movimento.Ordem_Filial = @Filial OR @Filial IS NULL)
        AND Movimento.Tipo_Operacao IN ('VND', 'VPC', 'VEF', 'DEV', 'CVE')
),

CONTAS AS (
    SELECT
        Financeiro_Formas_Recebimento_Pagamento_Contas_V2.Ordem,
        Financeiro_Formas_Recebimento_Pagamento_Contas_V2.Ordem_Formas_Recebimento_Pagamento,
        CASE Financeiro_Formas_Recebimento_Pagamento_Contas_V2.Tipo_Forma_Recebimento_Pagamento
            WHEN 'B' THEN 'Boleto'
            WHEN 'C' THEN 'Cartão de Crédito'
            WHEN 'D' THEN 'Cartão de Débito'
            WHEN 'H' THEN 'Cheque'
            WHEN 'N' THEN 'Carnê'
            WHEN 'R' THEN 'Carteira'
        END                                                                   AS Tipo_Recebimento,
        Financeiro_Formas_Recebimento_Pagamento_Contas_V2.Quantidade_Parcelas,
        Financeiro_Formas_Recebimento_Pagamento_Contas_V2.Valor_Total
    FROM Financeiro_Formas_Recebimento_Pagamento_Contas_V2
    INNER JOIN VENDAS
        ON Financeiro_Formas_Recebimento_Pagamento_Contas_V2.Ordem_Formas_Recebimento_Pagamento
           = VENDAS.Ordem_Financeiro_Formas_Recebimento_Pagamento_V2
)

SELECT
    CONVERT(VARCHAR(10), @Data_Inicial, 103)  AS Data_Inicial,
    CONVERT(VARCHAR(10), @Data_Final,   103)  AS Data_Final,
    Codigo_Cliente, Nome_Cliente, Nome_Vendedor,
    Codigo_Filial, Sequência_de_Venda,
    R.Tipo_Recebimento,
    R.Parcelas,
    R.Valor
FROM VENDAS
LEFT JOIN CONTAS
    ON CONTAS.Ordem_Formas_Recebimento_Pagamento = VENDAS.ORDEM
CROSS APPLY (VALUES
    (CONTAS.Tipo_Recebimento,  CONTAS.Quantidade_Parcelas, CONTAS.Valor_Total),
    ('Dinheiro',         1, VENDAS.Dinheiro_Valor),
    ('Cartão Convênio',  1, VENDAS.Cartao_Convenio_Valor),
    ('Pix',              1, VENDAS.Conta_Bancaria_Valor),
    ('Conta Caderno',    1, VENDAS.Conta_Caderno_Valor),
    ('Crédito',          1, VENDAS.Credito_Valor),
    ('Não Identificado', 1, VENDAS.Nao_Identificado_Valor),
    ('Produtos Trocados',1, VENDAS.Produtos_Trocados_Valor),
    ('Vale',             1, VENDAS.Vale_Valor),
    ('Troco',            1, VENDAS.Valor_Troco)
) AS R(Tipo_Recebimento, Parcelas, Valor)
WHERE
    R.Valor IS NOT NULL
    AND R.Valor <> 0
GROUP BY
    Codigo_Cliente, Nome_Cliente, Nome_Vendedor,
    Codigo_Filial, Sequência_de_Venda,
    R.Tipo_Recebimento, R.Parcelas, R.Valor
ORDER BY
    Codigo_Filial, Sequência_de_Venda,
    Codigo_Cliente, Nome_Cliente, Nome_Vendedor,
    R.Tipo_Recebimento, R.Parcelas, R.Valor
